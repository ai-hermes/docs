本指南将帮助您充分利用 Rethink AI 的各项功能。

## 聊天界面

### 发送消息

在底部输入框中输入您的问题，按 Enter 或点击发送按钮。AI Agent 将自动调用相关工具查询您的聊天数据，并以自然语言回复。

### Agent 模式切换

在输入框旁可以切换 Agent 模式：

- **Deep 模式**: 基于 DeepAgents 的深度推理模式，适合复杂分析任务，支持持久化记忆 (SQLite checkpointer)
- **React 模式**: 轻量级 ReAct Agent，适合简单的数据查询，响应更快

### 会话管理

- **新建会话**: 点击侧边栏的 "新建聊天" 按钮
- **切换会话**: 在侧边栏中点击历史会话
- **删除会话**: 右键或滑动删除不需要的会话
- **自动标题**: AI 会自动为每个会话生成标题

### 工具调用

当 AI 使用工具查询数据时，消息中会显示工具调用状态：

- 工具名称和执行状态 (运行中/已完成/失败)
- 点击工具状态可以在侧边栏查看调用详情 (输入参数和返回结果)

### 继续执行

当 Agent 因达到最大执行步数限制而停止时，您可以点击 "继续" 按钮让 Agent 从上次中断处恢复执行。

## 可用的 AI 工具

AI Agent 可以使用以下内置工具查询您的聊天数据：

- **获取最近会话** — 查看最近的聊天列表
- **获取聊天记录** — 查看指定联系人或群聊的消息历史
- **搜索消息** — 全文搜索聊天记录
- **获取联系人** — 查询联系人列表，支持过滤
- **获取联系人标签** — 查看标签/分组
- **获取标签成员** — 查看某个标签下的联系人
- **获取新消息** — 查看未读消息
- **获取聊天图片** — 提取聊天中的图片
- **解码图片** — 解密聊天图片文件
- **网络搜索** — 通过 Tavily 搜索网络信息 (需配置 API Key)

## 设置配置

点击左下角的设置按钮打开设置面板。

### LLM 供应商配置

- 支持配置多个 LLM 供应商
- 每个供应商需要填写: 名称、API Base URL、API Key
- 选择当前使用的供应商和模型
- 支持连通性检测

### LLM 引擎选择

在设置中可以切换底层 AI 引擎：

- **LangGraph** (默认): 支持 Deep/React 两种 Agent 模式
- **Claude Code**: 使用 Anthropic Claude SDK
- **Pi-AI**: 使用 Pi-AI 引擎，支持 MCP 工具桥接

### MCP 服务器配置

配置 MCP (Model Context Protocol) 服务器以扩展 AI 工具能力：

1. 在设置中找到 **MCP 服务器** 区域
2. 添加新的 MCP 服务器配置
3. 填写服务器名称、命令/URL、参数
4. 支持 stdio 和 SSE 两种连接方式

### IM 运行时配置

配置 Docker 容器运行 IM 客户端：

- Docker 镜像名称和标签
- VNC 连接端口
- 容器数据挂载路径

### 本地解密配置

配置本地聊天数据库解密：

- 聊天数据库文件路径
- 解密服务状态监控
- CDC 同步开关和配置

### Langfuse 可观测性 (可选)

为 AI Agent 执行过程添加追踪：

1. 启用 Langfuse 集成
2. 填写 Langfuse Public Key 和 Secret Key
3. 配置 Langfuse 服务器地址
4. 可在 Langfuse 面板中查看 Agent 执行链路

### 日志管理

- 设置日志级别 (debug/info/warn/error)
- 查看日志文件夹
- 导出日志文件用于问题排查

## 模型切换

在聊天界面顶部可以切换当前使用的 LLM 模型：

1. 点击模型切换器
2. 从已配置的供应商和模型列表中选择
3. 切换即时生效，无需重启

## 会话录制

Rethink AI 集成了 rrweb 会话录制功能：

- 应用会自动录制用户交互会话
- 可以在设置中查看和管理录制的会话
- 支持会话回放

## 自动更新

应用支持自动更新：

- 启动时自动检查更新
- 下载更新包 (支持差量更新)
- 可以在设置中关闭自动更新检查
